Founders and Coders Cheat Sheet

Git

  • clone [url]
  • add [filename] (add -A to do all files)
  • commit -m "Commit message"
  • push [origin master]
  • pull [origin master]
  • log (log --graph --oneline [branch 1] [branch2])
  • diff [commit ID old] [commit ID new] (no arguments staging area vs. working directory), diff --staged (staging area vs. repo)
  • checkout [commit ID you want] (checkout master takes you to the most recent)
  • init
  • status
  • reset (clears staging area), reset --hard resets staging area and working directory
  • branch (can make a new one by giving it an argument) (branch -d [branchname] to delete)
  • merge [branch1 (master)] [branch2] (if there is a conflict, you must go in to the file and correct it)
  • show [commit ID]
  • git remote add [origin]
  • fetch [origin master]
  • https://eu.udacity.com/course/how-to-use-git-and-github--ud775
  • remote [-v]

Markdown

  • _In Italics_
  • **In Bold**
  • # Header 1
  • ## Header 2
  • Please click through to this [link](www.github.com)
  • Here is a picture of an apple: ![A representation of an Apple](www.applepictures.com)
  • >Blockquotes are started with the ">" character
  • Lists are highlighted in the following way:
    • *Unordered list
    1. 1. Ordered list (type in "1. list item")
  • Hard paragraphs

    are done using two returns
  • Soft paragraphs
    are done using two spaces at the end of the line then one return

Bash

  • cd [.. or filename]
  • ls (can add -a to show hidden files)
  • pwd
  • mkdir [directory name]
  • mv [filename] (also renames the file)
  • CTRL + L - clear terminal
  • cp [filename] [directory and or copy file name]
  • rm [-r for folders] [file/folder name]
  • pushd [directory]
  • popd
  • touch [filename]
  • https://learnpythonthehardway.org/book/appendix-a-cli/introduction.html

Regular expressions

  • let reg = /abc/
  • let reg = new RegExp("\\b("+myname+")\\b", "gi")
  • reg.test("abcd")
  • reg.exec("acde")
  • /ab(c)/.exec("abc")
  • /[a-g]/
  • /abc/i
  • /abc/g (careful will keep a record of index)
  • /abc/y (careful will keep a record of index)
  • /\w/
  • /\d/
  • /\s/
  • /\W/
  • /./
  • /^/
  • /\b/
  • /abc?d/
  • /abc+/
  • /abc*/
  • /\d{1,4}/
  • /\babc\b/
  • /ab(c|d|e|)/
  • mystring.replace(/abc/,"test")
  • "Denis, Kent".replace(/(\w+), (\w+)/g,"$2 $1 $&")
  • "abc".replace(/ab/,str => str.toUpperCase())
  • mystring.replace(/\/\/.*|\/\*[^]*?\*\//g,"")
  • myname.replace(/[\\[.+*?(){|^$}]/g, "\\$&")
  • "test 1".search(/1/)
  • reg.lastIndex (after having done an exec)
  • mymatch.index (used on execed text)
  • mystring.match(/abc/g)
  • https://eloquentjavascript.net/09_regexp.html

CSS layout

  • display: block, inline, none, flex, inline block
  • position: static (default), relative, fixed, absolute (for absolute to work within the direct parent, parent must be position:relative)
  • float: right, left (make sure to clear the items below the float clear: right, left, both) (To fix overflow use overfloe:auto in the parent div)
  • @media screen and (min-width:600px), @media screen and (max-width:599px)
  • inline-block, vertical-align
  • display: flex (https://bocoup.com/blog/dive-into-flexbox)
  • flex-direction: row (default), row-reverse, column, column-reverse
  • justify-content: flex-start (default), flex-end, center, space-between, space-around (acts on the main axis-horizontal)
  • align-items: flex-start (default), flex-end, center, baseline, stretch (acts on the cross-axis - vertical)
  • flex-wrap: nowrap (default), wrap, wrap-reverse (acts on the cross axis)
  • align-content: stretch(default), flex-start, flex-end, center, space-between, space-around (acts on the main axis)
  • margin: auto (stretches the empty space for flex items)
  • align-self: stretch (default), flex-start, flex-end, center, baseline (overides align-items for individual flex items)
  • flex: [n] (free space for items), initial, none
